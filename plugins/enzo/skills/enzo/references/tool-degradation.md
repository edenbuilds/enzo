# Tool capability and graceful degradation

Map the need to any equivalent capability available in the environment:

| Need                | Preferred evidence               | Fallback                                    |
| ------------------- | -------------------------------- | ------------------------------------------- |
| Live page           | Browser inspection and page text | User screenshots or exported copy           |
| Visual design       | Desktop and mobile captures      | Supplied images                             |
| Responsive behavior | Multiple viewports               | Explicitly mark untested                    |
| Codebase            | File search and relevant source  | Supplied snippets or repository description |
| Competitors         | Current public research          | User-provided alternatives                  |
| Product flow        | Safe browser navigation          | Recording or flow description               |
| Documents           | Native document/PDF reading      | Pasted text or page images                  |

Continue with partial evidence. Never claim to browse, click, inspect, measure, or test a surface that was unavailable. Ask for the smallest artifact that would materially improve confidence.
