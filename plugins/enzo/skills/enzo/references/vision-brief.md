# Vision Brief and handoff

## Vision Brief

1. Project North Star
2. Intended use and commercial or utility context
3. Target audience and jobs-to-be-done
4. Positioning, category, promise, differentiation, and proof
5. Competitive opportunity
6. Core messages and narrative order
7. Brand personality, tone, and traits to avoid
8. Visual direction: layout, type, color, density, imagery, motion, responsiveness, accessibility
9. Information architecture in priority order
10. Product and interaction priorities: must-have, later, key states, mobile behavior
11. Relevant success metrics
12. Constraints, risks, dependencies, and unresolved decisions
13. One recommended next action

## Design/no-code handoff

Add page structure, reusable sections, component variants, CMS needs, breakpoints, interactions, motion limits, content dependencies, SEO, and analytics.

## Codebase handoff

Add routes, component architecture, tokens, data structures, state, integrations, accessibility, responsive behavior, loading/error states, analytics events, implementation order, and likely files.

## Recreation handoff

Add principles to preserve, functional parity, originality requirements, protected elements to avoid copying, identity differentiation, MVP order, and brand-confusion risk.

## Personal/internal handoff

Add simplest viable architecture, local/private options, cost constraints, maintenance, data ownership, backup/export, and workflows that can remain manual.
