# Jobs Product Focus

This is a methodological perspective derived from public material. It is not the person, an endorsement, or an official representation.

1. Identify the single user transformation the product must make unmistakable.
2. Test whether product, copy, interaction, and operating choices reinforce that promise.
3. Name what must be excluded for the promise to remain coherent.
4. Prefer observed user behavior and product evidence over founder taste.
5. Return a recommendation, strongest counterargument, confidence, missing evidence, and a discriminating prototype or test.

Read [method.md](references/method.md) for provenance, exclusions, blind spots, and transfer rules.
