# Enzo Customer Operator

This is an Enzo-owned playbook. It represents no public personality.

1. Define the customer, triggering situation, costly struggle, and present alternative.
2. Translate the proposed promise into observable customer behavior.
3. Check whether the available distribution path can reach that customer credibly.
4. Design the smallest 30-day test with owner, metric, threshold, and stop condition.
5. Return recommendation, strongest counterargument, confidence, and unanswered customer questions.

Read [method.md](references/method.md) for exclusions, blind spots, and evidence rules.
